## 3. 学术界的相关证据

### 3.1 Barrier Option 定价的已知问题

学术文献对 barrier option 定价的研究非常成熟：

| 研究发现 | 对我们策略的启示 |
|---|---|
| **GBM 假设不成立**（Cont, 2001; Aït-Sahalia, 2002） | 纯 GBM 模型系统性低估触及概率，因为忽略了跳跃和波动率聚集 |
| **隐含波动率优于历史波动率**（Carr & Wu, 2009） | 用 IV 替代 RV 可提升 5–15% 定价准确度 |
| **跳跃扩散是必要的**（Merton, 1976） | 对 near-barrier 市场，跳跃可使触及概率增加 20–50% |
| **波动率偏度存在**（Bakshi, Cao & Chen, 1997） | hit_above 和 hit_below 的概率不对称，需要 skew 修正 |

**结论：当前模型在学术标准下是"幼稚的"（naïve）。** 但这不意味着策略无效——如果市场参与者更幼稚，我们仍然可以盈利。

### 3.2 预测市场效率的研究

| 研究发现 | 启示 |
|---|---|
| **预测市场通常有效**（Wolfers & Zitzewitz, 2004） | 专业预测市场的价格通常是最好的概率估计 |
| **但存在局部无效**（Manski, 2006） | 小众市场、高复杂度事件、流动性不足时，价格可能偏离 |
| **Crypto barrier 是新兴市场** | 参与者结构（散户为主）可能导致定价错误 |

**核心问题**：Polymarket 的 crypto barrier 市场是"足够无效"还是"足够有效"？

## 4. 证伪 vs 证实的标准

### 4.1 证伪标准（什么情况下应该放弃）

| 标准 | 阈值 | 验证方法 |
|---|---|---|
| **校准度偏差** | 模型概率系统性地高估或低估 > 5% | Calibration Plot |
| **Paper P&L 为负** | 4 周累计净收益 < 0 | 模拟成交记录 |
| **无法成交** | > 50% 的候选机会实际无法成交 | Orderbook 深度验证 |
| **夏普比率过低** | 夏普 < 0.5（考虑费用后） | 周度收益序列 |
| **Regime 依赖** | 只在特定市场条件下有效 | 分时段归因分析 |

### 4.2 证实标准（什么情况下可以投入实盘）

| 标准 | 阈值 | 验证方法 |
|---|---|---|
| **校准度良好** | 模型概率与实际频率偏差 < 3% | Calibration Plot |
| **Paper P&L 为正** | 4 周累计净收益 > 2%（费用后） | 模拟成交记录 |
| **可执行率高** | > 70% 的候选机会可完整成交 | Orderbook 深度验证 |
| **夏普比率合格** | 夏普 > 1.0 | 周度收益序列 |
| **样本外稳定** | 不同波动率 regime 下均有效 | 分时段归因分析 |

## 5. 验证方案设计

### 5.1 Phase 1：模型修复（1–2 周）

**必须完成的改动**：

```python
# 1. 波动率输入：IV + GARCH + RV 加权
annual_vol = w1 * IV_ATM + w2 * GARCH_forecast + w3 * RV

# 2. 漂移项：ETF 资金流 + 资金费率信号
drift = f(ETF_flow, funding_rate)  # ∈ [-0.3, 0.3] annualized

# 3. 跳跃扩散：Merton Jump Model
dS/S = μ dt + σ dW + J dN(λ)

# 4. 波动率偏度：调整 hit_above vs hit_below 概率
if direction == "hit_above":
    model_p *= skew_adjustment(skew_25d)
```

**数据来源**（美国服务器上可用）：
- Deribit API：ATM IV + 偏度（免费）
- Binance/OKX API：资金费率 + OI（免费）
- Farside：ETF 每日净流入（免费）

### 5.2 Phase 2：Walk-Forward Paper Trading（4–8 周）

**操作流程**：

```
Day 0:  Scanner 运行 → 保存所有候选到 observations 表
Day N:  市场到期结算 → UMA Oracle 公布结果
Day N+1: 对比 model_p vs 实际结果 → 更新 calibration 数据库
```

**关键输出**：

1. **Calibration Plot**
   ```
   横轴：模型预测概率（如 0.62, 0.58, 0.71...）
   纵轴：实际发生频率
   理想：45度对角线（完美校准）
   偏差：系统性在对角线上方（高估）或下方（低估）
   ```

2. **P&L Attribution**
   ```
   总收益 = Edge 收益 + 模型误差 + 滑点损失 + 费用 + 运气
   如果 Edge 收益 < 模型误差 → 策略无效
   ```

3. **Regime Analysis**
   ```
   高波动期（VIX > 30） vs 低波动期（VIX < 20）
   趋势期（ETF 净流入 > $100M/日） vs 震荡期
   如果只在特定 regime 有效 → 需要 regime-switching 模型
   ```

### 5.3 Phase 3：判决（第 8 周）

**决策矩阵**：

| Calibration | Paper P&L | 判决 |
|---|---|---|
| 良好（< 3% 偏差） | 正收益 + 夏普 > 1 | ✅ **证实，投入小资金实盘** |
| 良好（< 3% 偏差） | 正收益 + 夏普 < 1 | 🟡 **改进仓位管理后再试** |

| 偏差（> 5%） | 任何 | 🔴 **证伪，放弃** |
| 任何 | 负收益 | 🔴 **证伪，放弃** |

### 6.2 Crypto Barrier 市场可能是真的低效

与传统金融市场不同，Polymarket 的 crypto barrier 市场有独特特征：

- **参与者结构**：散户为主，可能不理解 barrier option pricing
- **信息滞后**：价格波动快于市场参与者更新预期
- **流动性不足**：大订单无法消化，价格可能长期偏离
- **情绪驱动**：FOMO/FUD 导致极端定价

这些特征意味着：**即使模型是"幼稚的"，只要比市场参与者不那么幼稚，就可能盈利。**
