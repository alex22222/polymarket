# Polymtrade 临近到期交易备忘录

## 核心结论

临近到期的 BTC / ETH 预测市场，确实会随着截止时间接近而变得更确定，但这不是漏洞。

市场价格本身也会同步向真实概率收敛。真正的机会不是“越接近到期越确定”，而是：

```text
我们的真实概率判断 > 当前可买入价格 + 成本
```

换句话说，不是买确定性，而是买被低估的确定性。

## 为什么临近到期不是漏洞

以“BTC 在 05/25 12:00 是否高于某个价格”为例，越接近 12:00，剩余时间越短，价格穿越目标价的空间越小，因此 YES 或 NO 的真实概率会更接近 0 或 1。

但市场参与者也知道这一点。

如果还剩几分钟，BTC 明显高于目标价，YES 的真实概率可能是 99%。但此时盘口里的 YES 通常也会被报价到 98%、99% 甚至更高。此时买入虽然胜率高，但可赚空间很薄。

示例：

```text
真实概率: 99.0%
YES 买入价: 99.2%
结果: 负期望
```

看起来几乎必赢，但长期会亏。

## 80% 确定性是否值得买

80% 确定性本身不代表有钱赚。是否值得买，取决于买入价格。

假设模型判断 YES 胜率为 80%：

```text
YES 价格 0.60: edge 20%，理论 ROI 约 33%
YES 价格 0.75: edge 5%，理论 ROI 约 6.7%
YES 价格 0.80: 公平价，长期基本不赚钱
YES 价格 0.85: 负收益，长期亏钱
```

因此，值得行动的不是“80% 确定”，而是：

```text
模型概率 >= 80%
买入价格明显低于模型概率
spread 不大
可成交量足够
规则明确
剩余时间内价格距离 barrier 有安全垫
```

## 真正的机会在哪里

临近到期交易中，真正可利用的是市场错价，而不是时间本身。

可以用这个简化公式判断：

```text
edge = 模型概率 - 买入价格 - 成本
ROI = edge / 买入价格
```

一个可考虑的机会：

```text
模型概率: 80%
YES ask: 70%
成本/滑点: 1%
edge: 9%
ROI: 约 12.9%
```

一个不该做的机会：

```text
模型概率: 80%
YES ask: 81%
成本/滑点: 1%
edge: -2%
ROI: 负
```

## 为什么没有被轻松套利

临近到期的 BTC / ETH 市场很适合机器人交易，因此明显错价通常会很快被吃掉。

常见原因：

```text
1. 市场价格已经反映真实概率
2. 看似便宜的盘口数量很小
3. 下单时盘口已经撤单或移动
4. 买入会产生滑点
5. 临近结束时买卖盘可能很薄
6. 市场规则可能指定特定价格源、时间口径或结算方式
7. 结算不是即时到账，仍有规则解析和争议风险
```

所以这类机会不是无风险套利，更像是“信息、速度、执行和规则理解”的综合优势。

## Scanner 应重点判断什么

Scanner 不应该只看模型概率，而要同时看盘口。

关键字段：

```text
模型概率
YES / NO 最新 bid-ask
spread
order book 深度
可成交 notional
滑点
剩余时间
现价距离 barrier 的安全垫
规则和结算源
预期 ROI
```

一个合格候选应该满足：

```text
正 edge
正 ROI
可成交量足够
盘口不过宽
规则可解析
实时价格源可靠
```

## 自动化任务处理记录

检查自动化配置时发现两个任务高度重复：

```text
polymtrade-data-refresh
polymtrade-data-refresh-and-scanner
```

两者都会每小时运行一次 `polymtrade.research.automation_job`，并执行数据刷新、scanner、保存 observation 等流程。

重复运行会导致：

```text
重复写 scanner observation
重复调用外部 API
自动化健康记录混杂
候选复盘样本被重复放大
```

已处理：

```text
保留: polymtrade-data-refresh-and-scanner   ACTIVE
暂停: polymtrade-data-refresh               PAUSED
```

后续每小时只保留新的综合任务运行。

## 当前行动建议

下一步不要追求“临近到期看起来很稳”的手工下注，而应该继续强化 scanner：

```text
1. 用实时 BTC / ETH 价格判断距离 barrier 的安全垫
2. 用剩余时间波动率估算真实触及概率
3. 用最新 order book 判断能否以合理价格成交
4. 只保存正 edge、正 ROI、盘口质量足够的候选
5. 用 paper trading 跟踪这些候选是否真的长期赚钱
```

在样本量不足前，不建议真金白银放大仓位。
